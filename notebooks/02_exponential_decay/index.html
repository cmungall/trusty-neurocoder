    Learning a Scalar ODE Parameter via Cajal¶

    Exponential Decay: dC/dt = -k·C¶

    The simplest scientific surrogate: given observations of a decaying quantity, recover the rate constant k.

    Cajal program:

    iter{C₀ | c ↪ f(c)}(n)

    where f(c) = w·c is a learnable scalar multiplication.

    After training:

    • Learned weight w → recovered rate k = (1 - w) / dt
    • The structure (iteration, ODE skeleton) is fixed; only the rate is learned

    This is the "verified surrogate" idea in miniature: physics skeleton is correct by construction.

    How to Read This Notebook¶

    If you ignore the Cajal and PyTorch syntax for a moment, the story is simple:

    • start with an initial concentration C0
    • apply the same decay step over and over
    • adjust one unknown number until the predicted curve matches the observed curve

    In software terms, this is just:

    state = C0
for n in range(N_STEPS):
    state = step(state)

    The kernel here is the tiny update step(state). Cajal provides the repeated-application structure, while PyTorch learns the unknown scalar inside that step.

    1. Create Synthetic Data from Known Physics¶

    This first code cell sets up the ground truth.

    • We choose a true decay rate k = 0.3.
    • With timestep dt = 0.1, the one-step multiplier is w = 1 - k*dt = 0.97.
    • From that we generate the sequence C(0), C(1), ..., C(9).

    Those values play the role of observations. In a real use case they could come from lab data or from an expensive simulator that we want to approximate with a cheaper structured surrogate.

    class LearnableDecay(nn.Module):
    """f(c) = w·c  —  a single learnable scalar."""
    def __init__(self):
        super().__init__()
        self.w = nn.Parameter(torch.tensor([0.5], device=device))  # start deliberately wrong

    def forward(self, c):
        return TypedTensor(self.w * c.data, c.ty)


# Build Cajal program: iter{C₀ | c ↪ f(c)}(n)
program = TmIter(TmVar("c0"), "c", TmApp(TmVar("f"), TmVar("c")), TmVar("n"))
compiled = compile(program)

decay_fn  = LearnableDecay()
optimizer = torch.optim.Adam(decay_fn.parameters(), lr=0.01)
c0_tt     = TypedTensor(torch.tensor([C0], device=device), TyReal(1))

losses = []
w_history = []

for epoch in range(300):
    optimizer.zero_grad()
    total_loss = torch.tensor(0.0, device=device)

    for i in range(N_STEPS):
        n_onehot = torch.zeros(N_STEPS, device=device)
        n_onehot[i] = 1.0
        n_val = TypedTensor(n_onehot, TyNat())
        result = compiled({"c0": c0_tt, "f": lambda c, _fn=decay_fn: _fn(c), "n": n_val})
        total_loss = total_loss + (result.data[0] - cs_true[i].to(device)) ** 2

    total_loss.backward()
    optimizer.step()
    losses.append(total_loss.item())
    w_history.append(decay_fn.w.item())

w_final = decay_fn.w.item()
k_final = (1.0 - w_final) / DT
print(f"True k:      {TRUE_K}")
print(f"Recovered k: {k_final:.6f}")
print(f"Error:       {abs(k_final - TRUE_K):.6f}")

    2. Learn the Missing Number¶

    This cell defines the learnable part and runs training.

    • LearnableDecay contains exactly one trainable scalar: w.
    • TypedTensor is just the repo's typed wrapper around the numeric value.
    • TmIter(...) means: start at c0, then apply f repeatedly.
    • n is one-hot encoded because Cajal currently represents natural numbers that way.

    You do not need to know PyTorch internals to follow the idea. The optimizer is simply adjusting w so the predicted concentrations get closer to the observed ones.

    3. How to Read the Plots¶

    • Left: total prediction error over training. This should go down.
    • Middle: the learned per-step decay factor w. This should move toward the true value 0.97.
    • Right: the predicted decay trajectory should overlap the ground-truth curve.

    After training, we convert the learned discrete multiplier back into the scientific rate constant using:

    k = (1 - w) / dt

    That translation matters because scientists usually reason about the rate constant k, not the discrete step multiplier w.

    4. Verification in Plain Language¶

    The last code cell checks whether the learned result still behaves like a sensible decay process.

    We want:

    • 0 < w < 1, so each step is a stable decay step
    • C(n) >= 0, because negative concentration would be unphysical
    • the symbolic form to still read like the original scientific model

    This is the notebook-scale version of the Trusty Neurocoder claim: keep the structure fixed, learn the uncertain parameter, and then check simple scientific properties afterward.
